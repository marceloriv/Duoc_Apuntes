# Glosario

## A

- **Acceso no autorizado:** Ingreso a sistemas o datos sin permisos validos.
- **Active Directory (AD):** Servicio de Microsoft para administrar usuarios, equipos, permisos y politicas en redes corporativas.
- **Activo:** Recurso con valor para la organizacion que debe protegerse (informacion, sistemas, personas, procesos, infraestructura).
- **Amenaza:** Evento, actor o condicion capaz de causar dano a un activo.
- **Ataque de cadena de suministro:** Compromiso de un proveedor o componente externo para llegar al objetivo principal.
- **Autenticacion:** Proceso para verificar la identidad de un usuario o sistema.
- **Autenticacion multifactor (MFA):** Verificacion usando dos o mas factores (algo que sabes, tienes o eres).

## B

- **Backdoor:** Mecanismo oculto que permite acceso remoto persistente a un sistema comprometido.
- **Backup (respaldo):** Copia de seguridad de datos para recuperacion ante perdida, ataque o falla.
- **Blue Team:** Equipo defensivo que monitorea, detecta y responde incidentes de seguridad.
- **Broken Access Control:** Falla de control de acceso que permite a un usuario ver o modificar recursos de otro.

## C

- **C2 (Command and Control):** Canal de comunicacion entre un equipo comprometido y la infraestructura del atacante.
- **CRM (Customer Relationship Management):** Sistema para gestionar clientes, oportunidades comerciales, ventas y relacion con el cliente.
- **Ciberespionaje:** Operacion orientada a obtener informacion estrategica y mantener acceso oculto.
- **Ciberseguridad:** Disciplina para proteger sistemas, redes, datos y servicios digitales frente a ataques.
- **Cifrado:** Tecnica que transforma datos para que solo usuarios autorizados puedan leerlos.
- **Confidencialidad:** Pilar CIA que asegura que solo personas autorizadas accedan a la informacion.
- **Control correctivo:** Medida aplicada despues de un incidente para contener, recuperar y reducir impacto.
- **Control detectivo:** Medida orientada a identificar actividad anomala o maliciosa.
- **Control preventivo:** Medida orientada a evitar que ocurra un incidente.
- **Control de acceso:** Mecanismo que define que recursos puede usar cada identidad.
- **CVE (Common Vulnerabilities and Exposures):** Identificador estandar para una vulnerabilidad publica conocida.
- **CVSS v3 (Common Vulnerability Scoring System):** Sistema de puntuacion que mide la severidad de una vulnerabilidad.
- **CTI (Cyber Threat Intelligence):** Disciplina que analiza actores, campanas, TTPs e indicadores para anticipar amenazas.

## D

- **DDoS (Distributed Denial of Service):** Saturacion distribuida de un servicio para dejarlo fuera de linea.
- **Delivery:** Fase de la Kill Chain en la que el atacante entrega el vector de ataque (correo, enlace, archivo, etc.).
- **Deteccion de amenazas:** Identificacion de eventos sospechosos mediante monitoreo, logs y analitica.
- **Disponibilidad:** Pilar CIA que asegura acceso oportuno a informacion y servicios.
- **Domain Controller:** Servidor central de AD que valida autenticacion y autoriza accesos en el dominio.

## E

- **EDR (Endpoint Detection and Response):** Tecnologia que monitorea endpoints para detectar comportamiento malicioso y responder.
- **Escalacion de privilegios:** Tecnica para pasar de permisos limitados a permisos administrativos.
- **Escaneo de puertos:** Tecnica para identificar puertos abiertos y servicios expuestos en un sistema.
- **ERP (Enterprise Resource Planning):** Sistema integrado para administrar recursos y procesos de la empresa, como finanzas, inventario, compras y operaciones.
- **Exfiltracion de datos:** Extraccion no autorizada de informacion hacia infraestructura externa.
- **Exploitation (Explotacion):** Fase de la Kill Chain donde se aprovecha una vulnerabilidad o error para obtener acceso.

## F

- **Firewall:** Control de red que permite o bloquea trafico segun reglas definidas.
- **Fuga de informacion:** Exposicion o filtracion de datos sensibles a terceros no autorizados.

## G

- **Gestion de parches:** Proceso de aplicar actualizaciones de seguridad para corregir vulnerabilidades.
- **Gestion de riesgos:** Proceso de identificar, evaluar, priorizar y mitigar riesgos de seguridad.
- **Gobierno de seguridad:** Marco organizacional para dirigir y controlar decisiones, prioridades y responsabilidades de seguridad.
- **GRC (Governance, Risk and Compliance):** Enfoque que integra gobierno, gestion de riesgos y cumplimiento normativo.

## H

- **Hardening:** Reduccion de superficie de ataque mediante configuraciones seguras, cierre de puertos y eliminacion de servicios innecesarios.
- **Hash criptografico:** Resumen matematico de datos usado para verificar integridad.

## I

- **Indicador de compromiso (IoC):** Evidencia observable de una intrusion (hash, dominio, IP, patron de archivo, etc.).
- **Ingenieria social:** Tecnicas de manipulacion para inducir a usuarios a revelar informacion o ejecutar acciones inseguras.
- **Installation:** Fase de la Kill Chain donde el atacante instala malware o mecanismos de persistencia.
- **Integridad:** Pilar CIA que asegura que la informacion sea correcta y no alterada sin autorizacion.

## K

- **Kill Chain (Cyber Kill Chain):** Modelo que describe fases de un ataque desde reconocimiento hasta objetivo final.

## L

- **Log:** Registro de eventos de sistemas, aplicaciones o red, clave para auditoria y deteccion.

## M

- **Malware:** Software malicioso disenado para danar, robar informacion o tomar control de sistemas.
- **Movimiento lateral:** Desplazamiento del atacante dentro de la red tras el acceso inicial.

## N

- **NVD (National Vulnerability Database):** Base de datos publica de vulnerabilidades conocida y mantenida por NIST.
- **NIST Cybersecurity Framework:** Marco de seguridad basado en cinco funciones: Identify, Protect, Detect, Respond y Recover.

## P

- **Parche de seguridad:** Actualizacion que corrige una vulnerabilidad conocida.
- **Pentesting (Prueba de penetracion):** Simulacion controlada de ataque para encontrar y demostrar vulnerabilidades.
- **Persistencia:** Capacidad del atacante para mantener acceso en el tiempo tras comprometer un sistema.
- **Phishing:** Engano por correo o mensajeria para robar credenciales o distribuir malware.
- **Politica de seguridad:** Regla formal que define obligaciones y comportamiento esperado en seguridad.
- **Principio de minimo privilegio:** Asignar solo los permisos estrictamente necesarios para cada rol.
- **Privilegio:** Nivel de permiso que determina las acciones permitidas a una cuenta.
- **Puerto:** Numero logico que identifica un servicio de red en un host.

## R

- **Ransomware:** Malware que cifra sistemas o datos y exige pago para recuperar acceso.
- **RBAC (Role-Based Access Control):** Modelo de control de acceso basado en roles, donde los permisos se asignan segun funciones y responsabilidades.
- **RCE (Remote Code Execution):** Vulnerabilidad o tecnica que permite ejecutar codigo de forma remota.
- **Reconnaissance (Reconocimiento):** Fase de la Kill Chain donde se recolecta informacion del objetivo.
- **Red Team:** Equipo ofensivo que simula ataques reales para evaluar controles.
- **Riesgo:** Posibilidad de que una amenaza explote una vulnerabilidad y genere impacto sobre un activo.

## S

- **SIEM (Security Information and Event Management):** Plataforma para centralizar logs, correlacionar eventos y generar alertas.
- **SMB (Server Message Block):** Protocolo de Windows para compartir archivos e impresoras en red.
- **SOC (Security Operations Center):** Centro operativo donde se monitorea y responde a incidentes de seguridad.
- **Superficie de ataque:** Conjunto de puntos expuestos por donde un sistema puede ser comprometido.

## T

- **TCP:** Protocolo de transporte orientado a conexion, confiable y con entrega ordenada.
- **TTPs (Tacticas, Tecnicas y Procedimientos):** Formas de operar de actores de amenaza.

## U

- **UAC (User Account Control):** Mecanismo de Windows que solicita elevacion para acciones administrativas.
- **UDP:** Protocolo de transporte sin conexion, rapido y sin garantia de entrega.

## V

- **Vector de ataque:** Ruta o metodo utilizado para comprometer un sistema.
- **Vulnerabilidad:** Debilidad tecnica u operativa que puede ser explotada por una amenaza.

## W

- **WAF (Web Application Firewall):** Firewall especializado en filtrar trafico HTTP/HTTPS hacia aplicaciones web.
- **Weaponization:** Fase de la Kill Chain donde el atacante prepara payloads o artefactos maliciosos.
- **Worm (Gusano):** Malware que se propaga automaticamente entre sistemas.
