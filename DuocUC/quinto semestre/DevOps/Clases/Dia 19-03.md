# Día 19-03

- VPC(Virtual Private Cloud): red logica donde se define Ips y conectividad
- Subredes(Subnets): Son las divisiones de la VPS para separar las capas de servicio
- Internet Gateway: Es la conexion hacia Internet desde subredes públicas
- NAT Gateway: Salida a Internet desde subredes Privadas
- NACL:Habilita o bloque la salida a nivel de instancia

## Crear VPC

1. Crear VPC
   1. Deberia salir 1 vpc
   2. 2 zonas de disponibilidad
2. Configurar VPC
   1. VPC y mas
   2. El nombre del proyecto se reparte entre todos los elementos de la VPC
   3. Bloque CIDR IPV4
      1. Se compone entre la direcion ip y la mascara
      2. Del numero que se nos muestra tenemos 5 menos
      3. 10.0.0.0/20: Es la que usamos
   4. Bloque CIDR IPV6
      1. Sin Bloque CIDR
      2. Tenencia: Predeterminado
   5. Numero de zonas de disponibelidad
      1. El numero recomendado es 2
   6. Cantidad de subredes publicas
      1. Numero: 0
      2. Sin ninguna red publica, ya que primero se define los componentes privados
   7. Puerta de enlace NAT
      1. Se nos ofrece agregar en la lista blanca las conexiones
      2. Igualmente se asignan cargo
      3. Igualmente incrementa la superficie de ataque.
      4. Se coloco **Ninguna**
      5. Puntos de enlace de la VPC
         1. El profesor se creo con puerta de enlace Gatway s3
3. Luego se crea
